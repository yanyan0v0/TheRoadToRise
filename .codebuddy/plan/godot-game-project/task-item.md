# 实施计划

- [ ] 1. 搭建Godot项目结构与全局管理器框架
   - 创建 `project.godot` 配置文件，设置窗口分辨率1280×720、2D渲染模式、项目名称等基础参数
   - 创建核心目录结构：`scenes/`、`scripts/`、`resources/`、`assets/`、`data/`、`ui/`
   - 实现4个Autoload全局管理器脚本：
     - `GameManager.gd`：游戏状态枚举（MENU/CHARACTER_SELECT/MAP/BATTLE/EVENT/SHOP/REST/REWARD/GAME_OVER/VICTORY）及状态切换逻辑
     - `AudioManager.gd`：BGM/SFX独立音频总线、音量控制、淡入淡出播放
     - `SaveManager.gd`：JSON存档读写（user://目录）、版本号管理、错误处理
     - `EventBus.gd`：全局信号定义（角色属性变化、战斗事件、成就解锁等）
   - 实现场景切换工具类 `SceneTransition.gd`，支持淡入淡出过渡动画
   - _需求：1.1, 1.2, 1.3, 1.4, 17.2_

- [ ] 2. 实现数据配置系统与卡牌/敌人/法宝数据定义
   - 创建卡牌数据Resource类 `CardData.gd`（继承Resource），包含字段：card_id、card_name、card_type（ATTACK/SKILL/ULTIMATE）、rarity（COMMON/UNCOMMON/RARE/LEGENDARY）、energy_cost、description、effects数组、is_upgraded、character_exclusive
   - 创建敌人数据Resource类 `EnemyData.gd`，包含字段：enemy_id、enemy_name、enemy_type（NORMAL/ELITE/BOSS）、max_hp、intent_pattern、skills、phase_thresholds
   - 创建法宝数据Resource类 `RelicData.gd`，包含字段：relic_id、relic_name、rarity、trigger_type、effect_description、effects
   - 创建消耗品数据Resource类 `ConsumableData.gd`
   - 使用JSON文件定义所有卡牌数据（基础卡池+4个角色专属卡池，至少50张卡牌），包含设计文档中列出的所有示例卡牌
   - 使用JSON文件定义敌人数据（普通敌人、精英敌人、4个BOSS），包含白骨夫人等BOSS的多阶段数据
   - 使用JSON文件定义法宝和消耗品数据（紧箍圈、芭蕉扇、观音玉瓶、风火轮、镇妖塔等）
   - 实现 `DataManager.gd` 数据加载管理器，负责从JSON加载并缓存所有游戏数据
   - _需求：4.1, 4.2, 4.5, 7.4, 9.5_

- [ ] 3. 实现角色系统与角色选择界面
   - 创建角色数据类 `CharacterData.gd`，包含：character_id、character_name、max_hp、current_hp、mana、stamina（天蓬专属）、gold、armor、strength、passive_skill、starter_deck
   - 实现4个角色的初始化逻辑：

     - 卷帘大将：HP75/法力3/金币50，被动"每10点生命护甲+1"
     - 大唐圣僧：HP65/法力3/金币50，白龙马羁绊机制
   - 创建角色选择场景 `CharacterSelectScene.tscn`，展示4个角色卡片（含属性、被动技能描述），未解锁角色显示锁定状态和解锁条件
   - 实现角色属性变化信号机制，通过EventBus通知UI组件实时更新
   - _需求：3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 3.7_

- [ ] 5. 实现卡牌场景节点与手牌交互系统
   - 创建卡牌场景 `Card.tscn`（180×270px），包含：法力消耗圆形（左上角）、稀有度标识（右上角）、标题丝带（顶部15%）、卡牌插画区（中部45%）、类型标签、效果描述（底部40%）

     - 拖拽交互（跟随鼠标移动、目标区域高亮）

   - 实现卡牌动画：抽牌滑出翻转（0.3秒/张）、打出飞向目标（0.8秒）、弃牌滑入弃牌堆

- [ ] 6. 实现战斗系统核心逻辑与伤害/状态效果引擎
   - 创建战斗场景 `BattleScene.tscn`，布局包含：顶部状态栏（生命值/金币/牌组数/设置）、法宝栏、战斗区域（左侧角色+右侧敌人）、底部手牌区域、右下角"结束回合"按钮
   - 实现战斗状态机 `BattleManager.gd`，管理战斗流程：
     - 战斗开始：初始化护甲/状态、抽5张手牌、恢复法力
     - 玩家回合：抽牌、恢复法力、触发回合开始效果
     - 玩家行动：打出卡牌消耗法力、执行效果
     - 结束回合：手牌移至弃牌堆
     - 敌人行动：按意图执行攻击/防御/技能
     - 回合结束：结算状态效果、判定胜负
   - 实现伤害计算引擎 `DamageCalculator.gd`：
     - 公式：最终伤害 = 基础伤害 × (1+增伤%) × (1+易伤%) × (1-减伤%)
     - 护甲抵消 = min(护甲值, 最终伤害)，实际承受 = 最终伤害 - 护甲抵消
   - 实现状态效果系统 `StatusEffectManager.gd`：
     - 正面状态：金身、治愈、反甲、充盈、敏捷
     - 负面状态：破甲、减速、灼烧、封印、易伤
     - 状态图标+层数显示、回合结束按序结算
   - 实现战斗UI组件：角色/敌人血条、护甲值显示、状态效果图标栏、伤害飘字（红色伤害/绿色治疗/蓝色护甲）、敌人意图图标
   - _需求：5.1, 5.2, 5.3, 5.4, 5.5, 5.6, 5.7, 5.8, 5.9, 5.10, 6.1, 6.2, 6.3, 6.4, 6.5, 11.1, 11.5, 11.6, 11.7, 11.8_

   - 实现敌人召唤机制：战斗中动态添加新敌人到战场
   - 实现多敌人目标选择逻辑（玩家点击选择攻击目标）
