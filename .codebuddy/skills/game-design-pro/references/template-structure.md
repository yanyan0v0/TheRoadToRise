# 策划案模板结构规范

本文档定义了策划案各标准模块（Sheet/章节）的完整结构、内容要求和填写规则。

---

## 一、策划案标准模块总览

策划案由以下标准模块组成，每个模块对应Excel中的一个独立Sheet和Markdown中的一级章节：

| 模块 | Sheet名 | 必需 | 说明 |
|------|---------|------|------|
| 版本号 | 版本号 | ✅ | 方案元信息 |
| 交互说明 | 交互说明 / 交互 | ✅ | 美术资产评估、界面流转、UE说明 |
| 功能说明 | 功能说明 | ✅ | 核心功能规则描述 |
| 红点逻辑 | 红点逻辑 | ✅ | 红点触发与消除规则 |
| 配置需求 | 配置需求 | ✅ | 配置表字段定义 |
| IP&Icon需求 | IP&Icon需求 | ✅ | IP包装与Icon资源需求 |
| 埋点&GM需求 | 埋点&GM需求 | ✅ | 数据埋点与GM调试指令 |

以下模块为补充案模块，按需添加：

| 模块 | Sheet名 | 说明 |
|------|---------|------|
| 规则QA | 规则QA | 开发过程中细节规则的补充说明 |
| 版本改动 | 版本标识+改动内容 | 针对特定版本的功能变更记录 |
| 体验规划 | 体验规划（策划用） | 竞品对标、解锁时机、产出规划 |

## 二、版本号模块

### Markdown格式

```markdown
# 版本号

- **方案版本号**：1.0
- **设计人**：Apakoh(喻骋远)
- **方案内容**：[策划案名称]
- **概述**：[不超过50字的设计背景/目的/大概做法描述]
```

### Excel布局（版本号Sheet）

| 行 | 列B | 说明 |
|----|------|------|
| R2 | `方案版本号` | 标签（加粗12pt/合并B:D/蓝底白字） |
| R3 | `1.0` | 版本号值（11pt/合并B:D/浅底） |
| R5 | `设计人` | 标签 |
| R6 | `Apakoh(喻骋远)` | 默认作者 |
| R8 | `方案内容` | 标签 |
| R9 | `[策划案名字]` | 实际方案名 |
| R11 | `概述` | 标签（合并B:K） |
| R12 | `[概述内容]` | 概述文本（合并B12:K45，左上对齐+自动换行） |

### 内容规范

- 版本号从1.0开始，大改版时递增主版本号，小改动递增次版本号
- 设计人默认填写 `Apakoh(喻骋远)`
- 方案内容填写策划案的精确名称
- 概述需在50字以内，概括设计背景、核心目的和大致方案。格式参考：`通过[手段]，以[目的]。方案核心：[核心设计理念简述]`

## 三、交互说明模块

交互说明模块包含三个子区域，依次排列。

### 3.1 新增美术资产量评估

#### Markdown格式

```markdown
## 交互说明

### 新增美术资产量评估

| 分类 | 详细 | 数量 | 说明 |
|------|------|------|------|
| UI界面 | [界面名称] | 1 | [说明，如：独立界面prefab] |
| 弹窗 | [弹窗名称] | 1 | [说明，如：独立弹窗prefab] |
| Icon | [Icon名称] | 1 | [说明] |
```

#### Excel布局

| 行 | 列B | 列C | 列F | 列G |
|----|------|------|------|------|
| R2 | `交互说明`（Sheet大标题，合并B:U，深蓝底22pt） | | | |
| R4 | `新增美术资产量评估`（二级标题，合并B:L，蓝底14pt） | | | |
| R6 | `分类` | `详细`（合并C:E） | `数量` | `说明`（合并G:W） |
| R7+ | [类型] | [名称] | [数] | [说明] |

#### 资产分类规范

- **UI界面**：独立的全屏或半屏界面，每个界面单独一行
- **弹窗**：特殊弹窗单独一行，通用弹窗可合并标注数量
- **Icon**：功能Icon、道具Icon等，同类合并标注总数
- **视频/Timeline/2D动画**：按类型分行，同类合并
- **场景原画/3D资产**：特殊美术需求，单独枚举
- **不枚举项**：toast提示、已有资源复用

### 3.2 主要界面流转（概览）

#### Markdown格式

```markdown
### 主要界面流转（概览）

[描述所有界面之间的跳转关系和标准玩家操作流程]
[如有设计工具链接，贴上arthub/xd/figma URL]
[如有流转图，使用mermaid或文字描述流程]
主入口 → 列表/选择界面 → 详情界面 → 战斗/操作界面 → 结算界面
```

### 3.3 主要界面详细说明

#### Markdown格式

```markdown
### 主要界面详细说明

#### [界面名称A]

[界面的详细UE注解说明，包括：]
- 界面中各区域的功能说明
- 交互操作说明
- 状态变化说明
- 特殊表现说明
```

## 四、功能说明模块

### Markdown格式

```markdown
## 功能说明

### 玩法结构简介

[概述整个玩法/系统的结构，可包含流程图或示意图]

### [功能模块A名称]

#### [子功能A1名称]（若需要三级标题）

▸ [规则1描述]
    -- [子规则1.1]
    -- [子规则1.2]

▸ [规则2描述]
    -- [子规则2.1]
        -- [三级子规则]


### [功能模块B名称]

▸ [规则描述]
```

### Excel布局

| 行 | 说明 |
|----|------|
| R2 | `功能说明`（Sheet大标题，合并B:U，深蓝底22pt） |
| R4 | `玩法结构简介`（二级标题，合并B:L，蓝底14pt） |
| R5+ | 概述内容（从B列开始） |
| 空行 | |
| Rn | 二级子标题（合并B:L，蓝底14pt） |
| Rn+2 | 三级子标题（B列，加粗11pt，无标记符号） |
| Rn+3+ | 规则内容：B列=▸/--标记，C列=规则文本 |

### 内容规范

- **玩法结构简介**必须放在功能说明最前面
- 功能模块以**二级标题**划分
- 每个模块内的规则使用**▸/--层级标记**
- **段落之间空两行**
- 先写核心规则，再写边界情况/异常处理
- ⚠️ **设计范围约束**：功能说明只描述本系统自身的功能模块

### Excel布局

| 行 | 列 |

## 六、配置需求模块

### Markdown格式

```markdown
## 配置需求

### 新增[表名]表（[用途说明]）

| 字段名 | EN | 数据类型 | 打表类型 | 字段说明 |
|--------|-----|----------|----------|----------|
| [中文字段名] | [英文字段名] | int | 3 | [字段用途说明，含枚举值含义] |
| [中文字段名2] | [英文字段名2] | string | 2 | [字段用途说明] |

### [已有表名]字段拓展

| 字段名（新增） | EN | 数据类型 | 字段说明 |
|---------------|-----|----------|----------|
| [中文字段名] | [英文字段名] | int | [字段用途说明] |
```

### Excel布局

| 行 | 列 |
|----|-----|
| R2 | `配置需求`（Sheet大标题，合并B:O，深蓝底22pt） |
| R4 | 配置表名称标题（二级标题，合并B:L，蓝底14pt） |
| R6 | 表头行：字段名(B:C) \| EN(D) \| 数据类型(E) \| 打表类型(F) \| 字段说明(G:M) |
| R7+ | 字段数据行 |

### 数据类型规范

| 类型 | 写法 | 说明 |
|------|------|------|
| 整数 | `int` | 单个整数值 |
| 字符串 | `string` | 文本值 |
| 浮点数 | `float` | 小数值 |
| 整数数组 | `int[]` | 分号分隔的整数序列 |
| 布尔 | `bool` | 0/1 |
| 主键 | `int\|m` | 主键标记 |

### 打表类型规范

| 值 | 含义 |
|----|------|
| 2 | 客户端表 |
| 3 | 服务端表 |

### 配置表归属约束

- ⚠️ **只列出本系统直接管辖的配置表**
- 拓展外部表字段时标注 `*拓展字段`
- 引用外部表时使用一行引用说明

## 十、策划案类型与模块组合

### 10.1 系统/功能需求（【系统】前缀）

标准模块全套 + 可选补充案模块

- 交互说明可能较简单
- ⚠️ **边界提醒**：只设计本系统自身功能逻辑

### 10.2 养成线需求（【系统-养成线】前缀）

标准模块全套 + 详细配置需求

- 配置需求模块通常更详细，包含多张配置表和数值推演
- 可能包含"开发过程中遇到的一些问题及处理办法记录"Sheet
- ⚠️ **边界提醒**：只设计养成线自身的获取/升级/分解等规则

### 10.3 常驻玩法需求（【常驻玩法】前缀）

标准模块全套 + 体验规划

- 交互说明通常较详细，包含多个界面的UE说明
- 功能说明包含"玩法结构简介"子模块
- 经常包含版本迭代改动Sheet
- ⚠️ **边界提醒**：只设计玩法自身的规则和流程

### 10.4 活动玩法需求（【活动玩法】前缀）

标准模块全套

- 关注活动周期、开放时间、限时规则
- ⚠️ **边界提醒**：只设计活动玩法自身的规则

### 10.5 商业化方案（【商业化】前缀）

标准模块全套

- 关注付费点设计、定价策略
- ⚠️ **边界提醒**：只设计付费点和定价策略

## 十一、Markdown与Excel的对应关系

### 章节 ↔ Sheet 映射

| Markdown一级标题 | Excel Sheet名 |
|-----------------|--------------|
| # 版本号 | 版本号 |
| # 交互说明 | 交互说明 |
| # 功能说明 | 功能说明 |
| # 红点逻辑 | 红点逻辑 |
| # 配置需求 | 配置需求 |
| # IP&Icon需求 | IP&Icon需求 |
| # 埋点&GM需求 | 埋点&GM需求 |
| # 规则QA（补充） | 规则QA |

### 内容对应规则

1. Markdown中的**表格** → Excel中的**表格区域**（带表头样式）
2. Markdown中的**▸/--层级列表** → Excel中的**列偏移层级**（B列▸、C列--）
3. Markdown中的**二级标题** → Excel中的**合并单元格子标题**（蓝底14pt）
4. Markdown中的**三级标题** → Excel中的**加粗文本行**（B列11pt加粗）
5. Markdown中的**正文段落** → Excel中的**B列文本块**
6. Markdown中的**注释//行** → Excel中的**蓝色注释行**（theme=4蓝色）
7. Markdown中的**图片/流程图** → Excel中的**图片占位区域**（标注[图片占位: 描述]）
